\subsection{Grafische Benutzeroberfläche}

Die grafische Benutzeroberfläche dient der komfortablen Erzeugung von Automatendefinitionen.

\subsubsection{Zeichnen von Graphen}

\begin{description}
\item[/F0010/]
	\textit{Zustand erstellen:} Der Benutzer legt einen neuen Zustand im Graphen an, welcher ihm als ein Kreis angezeigt wird.
	Der Kreis beinhaltet einen Text, der dem Namen des Zustandes entspricht.
\item[/F0020/]
	\textit{Zustand löschen:} Bereits erstellte Zustände können vom Benutzer gelöscht werden.
\item[/F0030/]
	\textit{Zustand umbenennen:} Der Name eines Zustandes muss sich ändern lassen können.
\item[/F0040/]
	\textit{Zustand verschieben:} Die Position des Zustandes kann durch Verschieben angepasst werden.
\item[/F0100/]
	\textit{Übergang erstellen:} Zustandsübergange stellen die Aktionsmöglichkeiten des Automaten dar. Sie sind als Pfeile von einem Zustand zu einem anderen Zustand dargestellt.
\item[/F0110/]
	\textit{Übergang löschen}
\item[/F0120/]
	\textit{Übergang bearbeiten}
\item[/F0200/]
	\textit{Zustand zu Startzustand erklären:} Jeder Automat benötigt mindestens einen Startzustand, deterministische Automaten sogar genau einen. Dazu muss ein zuvor erzeugter Zustand in einen Startzustand umgewandelt werden können.
\item[/F0200/]
	\textit{Zustand zu Endzustand erklären}
\item[/F0200/]
	\textit{Startzustand oder Endzustand zu normalen Zustand erklären}
\end{description}

\subsubsection{Dateioperationen}

\begin{description}
\item[/F0300/]
	\textit{Automatendefinition speichern:} Um später wieder auf eine bereits erstellte Automatendefinition zurückgreifen zu können, wird diese im XML-Format abgespeichert.
\item[/F0310/]
	\textit{Automatendefinition laden:} Aus einer existierenden XML-Datei wird ein Automatengraph erzeugt.
\item[/F0320/]
	\textit{Neue Automatendefinition erstellen:} Der aktuell bearbeitete Graph wird verworfen.
\end{description}

\subsubsection{Simulation}

\begin{description}
\item[/F0400/]
	\textit{Simulation ausführen:} Die Benutzeroberfläche ruft mit dem aktuell angezeigten Automaten und dem Eingabewort die Simulationsfunktion der Bibliothek auf.
\item[/F0410/]
	\textit{Simulationsergebnis anzeigen:} Nach erfolgter Simulation wird das Ergebnis der Simulation, insbesondere das Ergebnis des Akzeptanztests, angezeigt.
\end{description}


\subsection{Kommandozeilenprogramm}

Neben der grafischen Benutzeroberfläche existiert ebenfalls ein Kommandozeilenprogramm, das über Parameter die Simulation eines Automaten anstößt.

\begin{description}
\item[/F1000/]
	\textit{Ausgabe einer Beschreibung der Programmparameter:} Um auf der Kommandozeile ausgeführt werden zu können benötigt das Simulationsprogramm eine Reihe von Parametern (Eingabewort, Automatendefinition, evtl. aus Datei gelesen). Diese Parameter werden mitsamt einer Beschreibung auf der Konsole ausgegeben.
\item[/F1010/]
	\textit{Ausführung der Simulation:} Nach Eingabe der erforderlichen Parameter ruft das Kommandozeilenprogramm die Bibliotheksfunktion zur Simulation des Automaten auf und zeigt das Ergebnis an.	
\end{description}

\subsection{Bibliothek}

Die Bibliothek ist eine separate Komponente, die unabhängig von der grafischen Benutzeroberfläche operiert. Sie führt die Simulation aus, tritt für den Benutzer jedoch selbst nicht in Erscheinung, da die Ergebnisse der Simulation an aufrufende Programme weitergereicht werden, damit diese das Ergebnis ihrerseits in gewünschter Form darstellen.
\\\\
Die Auslagerung des Simulationsvorgangs in eine Bibliothek wird gewählt, um anderen Projekten die Funktionalität zur Verfügung zu stellen.
